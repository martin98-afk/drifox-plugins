# Code Reviewer - DriFox 插件

为 DriFox 提供自动化代码审查能力，基于系统性 checklist 对 PR 或代码变更进行审查，输出质量评分和改进建议。

## 功能

| 功能 | 说明 |
|------|------|
| 🤖 **智能审查** | 基于多维度 checklist 自动审查代码变更 |
| 📊 **质量评分** | 从安全性、正确性、性能、可维护性等维度评分 |
| 📋 **审查清单** | 提供可操作的审查 checklist，逐项检查 |
| 🔍 **问题定位** | 精确定位问题文件和行号，给出修复建议 |
| 📝 **报告生成** | 生成结构化的审查报告，便于追踪 |

## 安装

### 方式一：复制到插件目录

```bash
# 复制插件到 DriFox 插件目录
cp -r plugins/code-reviewer ~/.drifox/plugins/

# 或在 Windows 上
xcopy /E /I plugins\code-reviewer %USERPROFILE%\.drifox\plugins\code-reviewer
```

### 方式二：从插件市场安装

```bash
/plugin --install code-reviewer
```

DriFox 启动时自动发现并加载插件。

## 使用方式

### 方式一：使用命令

```bash
# 审查当前未提交的变更
/review

# 审查暂存区内容
/review --staged

# 审查指定文件
/review --file=src/main.py

# 全项目审查
/review --full

# 查看审查 checklist
/checklist
```

### 方式二：使用智能体

```bash
# 派生审查智能体进行深度审查
@reviewer
```

### 方式三：技能自动注入

当涉及以下场景时，Code Reviewer 技能自动注入上下文：
- 代码审查、PR review
- 质量检查、安全审计
- 审查 checklist 相关问题

## 审查维度

| 维度 | 优先级 | 检查项 |
|------|--------|--------|
| 安全性 | 🔴 Critical | SQL 注入、XSS、命令注入、敏感信息泄露 |
| 正确性 | 🟠 High | 逻辑错误、错误处理、边界条件、并发安全 |
| 性能 | 🟠 High | N+1 查询、重复计算、阻塞操作 |
| 可维护性 | 🟡 Medium | 代码重复、函数过长、命名不清 |
| 风格一致性 | 🟢 Low | 格式化、命名风格 |
| 测试覆盖 | 🟡 Medium | 测试缺失、测试质量 |

## 审查报告示例

```
## 审查报告

**审查范围**: src/auth.py
**变更行数**: +45/-12
**审查时间**: 2026-06-26 14:00

### 摘要
| 严重程度 | 数量 | 说明 |
|---------|------|------|
| 🔴 严重 | 1 | 必须修复 |
| 🟡 中等 | 3 | 应修复 |
| 🟢 建议 | 2 | 可选优化 |

### 详细发现

#### 🔴 [P0] SQL 注入风险
- **文件**: `src/auth.py`
- **位置**: 第 23 行
- **问题**: 使用字符串拼接构建 SQL 查询
- **影响**: 可能被 SQL 注入攻击
- **建议**: 使用参数化查询

#### 🟡 [P1] 敏感信息硬编码
- **文件**: `src/auth.py`
- **位置**: 第 5 行
- **问题**: API 密钥硬编码在代码中
- **建议**: 使用环境变量

### 总结
本次审查发现 1 个严重问题和 3 个中等问题，建议在合并前修复严重问题。
```

## 工作原理

```
用户触发审查
    │
    ▼
DriFox 加载 code-reviewer 插件
    │
    ▼
执行 git diff / 读取指定文件
    │
    ▼
按审查维度分析代码
    │
    ▼
输出结构化审查报告
```
