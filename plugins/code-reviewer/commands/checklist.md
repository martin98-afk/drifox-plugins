# /checklist 命令 — 代码审查 Checklist

你正在处理 `/checklist` 命令，向用户展示代码审查的完整 checklist。

## 输出要求

输出一个结构化的代码审查 checklist，包含以下部分：

### 开审前 Checklist

- [ ] 了解需求背景和设计意图
- [ ] 确认测试覆盖情况
- [ ] 检查 CI 是否通过

### 安全性检查（Critical）

- [ ] 无硬编码敏感信息（密钥、Token、密码）
- [ ] 输入验证充分（类型、长度、格式）
- [ ] 权限检查正确（认证、授权）
- [ ] 无危险函数调用（eval、exec、shell 拼接）
- [ ] 无 SQL 注入风险（参数化查询）
- [ ] 无 XSS 风险（输入转义）
- [ ] 无命令注入风险

### 正确性检查（High）

- [ ] 业务逻辑符合需求
- [ ] 错误处理完善（异常捕获、错误传播）
- [ ] 边界条件覆盖（空值、零、极大值）
- [ ] 并发场景安全（竞态条件、资源竞争）

### 性能检查（High）

- [ ] 无 N+1 查询问题
- [ ] 无重复计算
- [ ] 无阻塞 I/O 操作
- [ ] 内存使用合理（无泄漏、无大对象）

### 可维护性检查（Medium）

- [ ] 代码无重复（DRY 原则）
- [ ] 函数长度合理（建议 < 50 行）
- [ ] 命名清晰有意义
- [ ] 注释充分（复杂逻辑有解释）
- [ ] 模块职责单一

### 风格一致性（Low）

- [ ] 符合项目格式规范
- [ ] 命名风格统一（camelCase/snake_case）
- [ ] 导入顺序正确

### 测试覆盖（Medium）

- [ ] 核心逻辑有测试
- [ ] 边界条件有覆盖
- [ ] 测试可读可维护

## 输出格式

使用 Markdown 格式，以 checkbox 形式呈现上述 checklist。

## 附加说明

在 checklist 后添加：

```
💡 提示：可以使用 /review 命令进行自动化代码审查
```
